## Process

A processor is a function `Process( ...args:any[] ) any` that performs a specific function, consisting of the processor name, parameter list, and return value. Processes include the following types:

| Process                 | Description                                           |
| ------------------------- | ----------------------------------------------------- |
| xiang.\*.\*               | Built-in processor                                    |
| models.<model name\>.\*   | built-in processes, atomic operations on data models |
| flows.<data flow name\>   | built-in handlers, handlers written with data flows   |
| scripts.<script name\>.\* | Built-in handlers, scripting handlers via JavaScript  |
| plugins.<plugin name\>.\* | Write processes via GRPC plugins                     |
| session.\*                | built-in, session data handler                        |

<Notice type='warning'>
	Note: App Engine built-in processor namespace is <strong>xiang</strong> , Prevent when used on
	the command line with
	<strong>yao</strong> confusion. <strong>xiang</strong> is the internal R&D code of the application
	engine, named after Zhouyi
	<strong>The Legend of the Elephant[xiàng zhuàn]</strong>, used to interpret hexagrams and lines,
	take the meaning of observing Vientiane, and predicting trends.
</Notice>

## How To Use

### Use In CLI

The processor `yao run <process> [args...]` can be run using the `run` command.

```bash
yao run models.pet.Get '::{}'
```

### Use In API

Specify `process` in the API to set the processor. `query` sets the parameter list for the handler.

```json
{
  ...
  "paths": [
    {
      "path": "/search",
      "method": "GET",
      "process": "models.pet.Paginate",
      "query": [":query-param", "$query.page", "$quey.pagesize"],
      "out": {
        "status": 200,
        "type": "application/json"
      }
    }
  ]
  ...
}
```

### Used In Dataflow Nodes

The processor is specified by `process` in the data flow node, and `args` is passed in the processor parameter table.

```json
{
  ...
  "nodes": [
    {
      "name": "pet",
      "process": "models.pet.Find",
      "args": [1, { "select": ["id","name"] }]
    },
    {
      "name": "print",
      "process": "yao.sys.Print",
      "args": ["{{$res.pet}}"]
    }
  ]
  ...
}
```

The same method can also be used on the data flow nodes of Kanban, large screen, and graph.

```json
{
  ...
  "nodes": [
    {
      "name": "Type summary",
      "process": "flows.stat.kind",
       "args": [["dog","cat"]]
    }
  ]
  ...
}
```

### Use In Data Tables

Use `process` to overload tabular data reading logic

```json
{
  ...
  "apis": {
    "search": {
      "process": "flows.pet.latest"
    },
    "find": {
      "process": "flows.pet.find"
    }
  }
  ...
}
```

Used as a form `Hook` to handle contextual data

```json
{
  ...
  "hooks": {
    "after:search": "flows.pet.fliter.data",
    "after:find": "flows.pet.fliter.row"
  }
  ...
}
```
